# Сайт-портфолио с системой сообщений

Современный веб-сайт портфолио с интегрированной системой обмена сообщениями в реальном времени.

## Структура проекта

```
portfolio-messaging-system/
├── frontend/           # Клиентская часть
│   ├── css/           # Стили
│   ├── js/            # JavaScript файлы
│   ├── images/        # Изображения
│   └── *.html         # HTML страницы
├── backend/           # Серверная часть
│   ├── server.js      # Основной сервер
│   ├── services/      # Бизнес-логика (будет добавлено)
│   └── routes/        # API маршруты (будет добавлено)
├── database/          # База данных
│   └── schema.sql     # Схема БД
└── package.json       # Зависимости проекта
```

## Требования

- Node.js 18+ (рекомендуется Node.js 20 LTS)
- npm или yarn
- **Важно**: Если у вас Node.js 24+, могут возникнуть проблемы с компиляцией better-sqlite3. Рекомендуется использовать Node.js 20 LTS.

## Установка

1. Клонируйте репозиторий или скачайте файлы проекта

2. Установите зависимости:
```bash
npm install
```

3. Инициализируйте базу данных:
```bash
npm run init
```

Это создаст базу данных SQLite и администратора по умолчанию:
- Email: admin@example.com
- Пароль: admin123

⚠️ **ВАЖНО**: Измените пароль администратора после первого входа!

4. (Опционально) Создайте файл `.env` на основе `.env.example` и настройте переменные окружения

## Запуск

### Режим разработки (с автоперезагрузкой):
```bash
npm run dev
```

### Продакшн режим:
```bash
npm start
```

Сервер будет доступен по адресу: http://localhost:3000

## Устранение неполадок

### Ошибка компиляции better-sqlite3

Если при установке зависимостей возникает ошибка компиляции better-sqlite3:

**Решение 1**: Используйте Node.js 20 LTS вместо Node.js 24+
```bash
# Установите Node.js 20 LTS с официального сайта nodejs.org
# Затем переустановите зависимости
npm install
```

**Решение 2**: Установите необходимые инструменты сборки для Windows
- Установите Visual Studio Build Tools с поддержкой C++
- Убедитесь, что установлен Python 3.x

**Решение 3**: Используйте альтернативную базу данных
- В будущих версиях можно заменить better-sqlite3 на другую библиотеку (например, sql.js)

### Проблемы с портами

Если порт 3000 уже занят, измените его в файле `.env`:
```
PORT=3001
```
